  1. Que es un sistema de control de versiones?

    Es un sistema que controla y organiza todos los cambios realizados en un archivo o conjunto de estos a lo largo del tiempo, de modo que facilita la recuperación de versiones especificas, de una manera mas fácil permite movernos en la linea de tiempo del archivo o conjunto.

  2. Cual es la diferencia entre git y github?

    Git es utilizado para trabajar de manera local pero cuando se desea colaborar con otras personas de manera remota, utilizar una interfaz web o publicar tus proyectos se utiliza Github es un sistema que guarda nuestro proyectó

  3. Cómo creo una rama en git?

    Con el comando git branch nombre_de_la_rama

  4. Cómo clono un repositorio?

    git clone URL/name.git NombreProyecto

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    git fell git pull

  6. Cómo puedo gener un tag?

    git tag "numero de la version" -m "mensage del tag"

  7. Cómo hago una mezcla de ramas?

    git mergue [rama_para_combinar]

  8. Cómo puedo moverme a un commit especifico?

    git log [id-del-commit]

  9. Cómo puedo borrar una rama?

    $ git branch -D nombre_de_la_rama

  10. Cómo puedo borrar un tag?

    git tag -d "numero de la version"

  11. Cómo puedo ver los cambios que estan por enviarse al repositorio?

    git status

  12. Para que sirve el comando git remote -v?

    Ver en el repositorio al que estas conectado

  13. Cómo le digo a git cual es mi email y correo?

    git config --global user.email "email"

  14. Cuales son los pasos para conectar git con github?

    Generar un repo extraer la URL git remote add origin "URL"
